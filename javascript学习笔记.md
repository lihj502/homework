## 一、javascript的初印象

和c语言的不同，js不需要编译，运行过程中由js解释器（引擎）来逐行的进行解释并运行，这样就感觉方便多了。

**tip:** js引擎（逐行解释js代码）和渲染引擎（翻译HTML和css）组成了浏览器。

但是js的大多数语法都可以找到c语言语法的影子，他们在很多地方有着有相同的特性

比如，js的每一条语句都需要分号的结尾

而且，随着js的发展，它的应用领域也是从一开始的进行简单的表单校验到了如今广泛的应用。

## 四、js的定义和声明类型的方法

并不类似于c语言需要一个特别的定义区分整数型和浮点数型，在js语言中，只有一个统一的var来完成各种不同赋值的变量定义

**对变量定义的方式:**

* 可以先仅对变量声明,之后再单独的赋值

* 也可以在声明的同时赋值

  推荐在声明的同时赋值,这样不容易忘记赋值。
  
  忘记赋值以后，变量是undefined

* 当然，也可以通过“，”在一个声明中用var对多个变量同时分别定义

  但是不可以同时定义多个变量的同时只用一个“=”赋同一个值

**注意:** js对大小写是敏感的，因此要注意大小写，它们的意义不同。

所定义的变量最后用字母开头，虽然可以用符号开头，但是没有具体的意义，因此不推荐这个方式。

**同时，js有着它独特的声明变量类型的方式:**

通过在定义变量后，用等于号后面使用关键词“new”后面添加类型来声明它的类型

和c语言不同，在c语言中如果重复定义一个变量，将会出现错误，但是在js中，可以重新声明 JavaScript 变量，如果第二次声明变量而没有赋值，则该变量的值不会丢失，如果第二次重新赋值，那么第一次的赋值将会被掩盖。

可以通过变量赋值Nell来清空变量



## 五、js中的数组

**语法规则:**

1. 可以在定义完数组以后以此给每一位赋值

   此方法的语法规则为：
   
   var cars=new Array();
   cars[0]="Saab";
   cars[1]="Volvo";
   cars[2]="BMW";
   
   。。。。。。

2. 在定义完数组的同时赋值

   此方法的语法规则为：
   
   var cars=new Array("Saab","Volvo","BMW");

**tip:** 数组的下标和c语言一样也是从零开始的，但是它不用定义数组的大小



## 六、js的对象

对象也是一个变量，但是可以通过大括号包含多个变量，并且要给每一个变量赋值，因此可以将js的对象看作变量的容器。

在对象中，每一个变量都是以键值对的形式出现

**键值对:** 语法规则 name：value,这样存在于变量中的键值对又被称为变量属性

### 6.1 分辨：什么是访问对象的方法

js方法是对象中表现的行为。js方法是包含函数定义的属性，即方法是有着对象属性的函数。



## 七、js函数

js函数是由事件驱动或者当它被调用时执行的可重复使用的代码块，其中事件是指发生在 HTML 元素上的事情，一般来说当在 HTML 页面中使用 JavaScript 时， JavaScript 就可以触发这些事件。

js函数的语法就是一段包裹在花括号中的代码块，前面使用了关键词 function：

**函数的调用:**

在调用函数的同时可以向其传递参数值，而这些参数值可以在函数中使用。

参数一次发送的参数值数量任意，但要记得用逗号隔开

**语法规范:** myFunction(*argument1,argument2*)

如果向函数中输入函数值的话，一定要在声明函数的同时，将参数作为变量来声明。

**语法规范:** function myFunction(***var1***,***var2***){	代码	}

**tip:** 变量和参数必须以相同的顺序在调用的位置以及声明的位置出现。

和c语言相同，js的函数同样可以设定返回值，在return出现以后，函数会停止执行，并返回指定的值，而这时调用函数的位置将被函数的返回值取代。

**PS:** 即使仅仅希望退出函数时 ，也可使用 return 语句，不过此时不需要添加额外的返回值。

在函数中的很多地方，js的语法和c语言都十分的相同，再比如，在 JavaScript 函数内部声明的变量（使用 var）就是局部变量，所以只能在函数内部访问它

根据局部变量的特点我们完全可以在不同的函数中使用名称相同的局部变量，因为只要函数运行完毕，本地变量就会被删除。

那么说到局部变量，与之相对的一定是全局变量，在函数外声明的变量被称作全局变量，网页上的所有脚本和函数都能访问它，在页面关闭以后，变量将会自动删除。

非严格模式下即使是在函数中给未声明变量赋值依然可以创建全局变量，而该变量将被自动作为 window 的一个属性，可以用delet删除。

# html，css，js各自职能和关系

三者各自的职能如下表

| 标准 |                           说明                            |
| :--: | :-------------------------------------------------------: |
| 结构 |        结构用于对网页元素进行整理和分类——HTML负责         |
| 表现 | 表现用于设置网页元素的版式、颜色、大小等外形样式——css负责 |
| 行为 |        行为是指网页模型的定义及交互的编写——js负责         |

三者之间的关系我想到了一个比较有意思的比喻（来自从小学追火影的老火影迷脑洞）

可以将三者的关系比喻为火影中佐助的须佐能乎的三种形态：

* 第一形态——html

  “此时的半身须佐能乎只有简单的骨架以及肌肉，处于较为原始的状态

* 第二形态——css

  此时的半身须佐能乎出现了皮肤和盔甲，对第一形态做了外观上的修饰

* 第三形态——js

  此时的须佐能乎变为完全体，可以站立行动，完成了从静到动的转化

总结来说，只有在html，css和js三者的共同发挥作用下，才会出现我们现在看到的丰富绚丽的各式网站。
